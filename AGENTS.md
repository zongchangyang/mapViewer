# KenyaMap — Agent & Developer Guide

## Project Overview

**KenyaMap** is a geospatial web application that visualizes large-scale raster datasets over Kenya using Leaflet.js. It serves two map types:

1. **LULC Map** — categorical land use/land cover classifications (2017-2025) from multiple providers
2. **Similarity Map** — continuous-value change detection between 2017 and 2025 using AlphaEarth embeddings

Data is served from local Cloud Optimized GeoTIFF (COG) files via `localtileserver`. The frontend layers these rasters on top of ESRI World Imagery and OpenStreetMap basemaps.

## Data Specifications

### LULC Datasets (Categorical)

All LULC rasters are single-band UInt8 COGs in EPSG:4326. Class names, palettes, and label mappings are defined in `dataset_label_mapping.json`.

| Dataset Key | Source | Resolution | Years Available | Classes | Notes |
|---|---|---|---|---|---|
| `sunstone_kenya_lulc_9C` | Sunstone Institute | 10m | 2017-2025 | 10 (incl. NoData) | Primary dataset |
| `dynamicworld` | Google/WRI | 10m | 2017-2025 | 9 | Near real-time LULC |
| `esri_lulc` | Impact Observatory/ESA | 10m | 2017-2024 | 9 | Sentinel-2 derived |
| `worldcover` | ESA | 10m | 2020-2021 | 12 | Non-sequential original values (10,20,...100) |
| `glad_glclu` | UMD GLAD | 30m | 2020 | 113 full / 11 simplified | Use `groups` for display |
| `glc_fcs30d` | Tsinghua University | 30m | 2019-2021 | 36 full / 13 simplified | Use `groups` for display |

**Important:** For `glad_glclu` and `glc_fcs30d`, always use the `groups` array from `dataset_label_mapping.json` for visualization. The full class sets are too granular for map legends.

### Similarity Dataset (Continuous)

| Property | Value |
|---|---|
| File | `alphaearth_similarity_2017_2025_30m_uint16_cog.tif` |
| Band | 1 (grayscale, UInt16) |
| Value range | 14-9,997 (p99 ~ 3,555) |
| Semantics | Higher value = less similar = more land change |
| Display range | 0-7,200 (covers ~99.9th percentile) |
| Resolution | 10m |
| CRS | EPSG:4326 |

### Reading dataset_label_mapping.json

The JSON file is keyed by dataset type. Each entry provides:

```
{
  "dataset_key": {
    "name": "...", "description": "...", "source": "...", "resolution": 10,
    "classes": [
      { "value": <raw_pixel_value>, "name": "...", "color": "#hex" },
      ...
    ],
    "groups": [                               // only for glad_glclu and glc_fcs30d
      { "name": "...", "color": "#hex", "members": [<raw_pixel_value>, ...] },
      ...
    ]
  }
}
```

When rendering a LULC tile:
1. Read pixel value from the raster
2. Look the value up directly in `classes` (or `groups.members` for grouped datasets) to get `name` and `color`

For datasets with `groups`, the `groups` block takes precedence for display (legend, tile colorization, area stats): multiple raw values collapse into a single group with one name and color. The full per-class `classes` list is kept as reference metadata.

## Implementation Guide

### 1. Tile Server Setup

Use `localtileserver` to serve COG files as XYZ tile endpoints. Each raster gets its own tile client.

```python
from localtileserver import TileClient, get_leaflet_tile_layer

# For LULC (categorical) — use nearest-neighbor resampling to preserve class values
lulc_client = TileClient("data/sunstone_kenya_lulc_2017_9Classes_assembleV1_cog.tif")

# For similarity (continuous) — default bilinear is fine
sim_client = TileClient("data/alphaearth_similarity_2017_2025_30m_uint16_cog.tif")

# Get tile URL template for Leaflet/folium
tile_url = lulc_client.get_tile_url()  # → http://localhost:{port}/tiles/{z}/{x}/{y}.png
```

For categorical rasters, apply a colormap server-side using the palette from `dataset_label_mapping.json`, or serve raw values and colorize client-side.

### 2. Build the Folium Map

```python
import folium
from localtileserver import get_leaflet_tile_layer

# Kenya center coordinates
KENYA_CENTER = [0.0236, 37.9062]
KENYA_ZOOM = 6

m = folium.Map(location=KENYA_CENTER, zoom_start=KENYA_ZOOM)

# Basemap: ESRI World Imagery
folium.TileLayer(
    tiles="https://server.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile/{z}/{y}/{x}",
    attr="Esri",
    name="ESRI World Imagery",
    overlay=False,
).add_to(m)

# Basemap: OpenStreetMap (default)
folium.TileLayer(
    tiles="openstreetmap",
    name="OpenStreetMap",
    overlay=False,
).add_to(m)

# Add raster overlay from localtileserver
lulc_layer = get_leaflet_tile_layer(lulc_client, name="LULC 2017", overlay=True)
lulc_layer.add_to(m)

# Layer control
folium.LayerControl().add_to(m)

m.save("map.html")
```

### 3. LULC Legend & Colorization

Build the legend dynamically from `dataset_label_mapping.json`:

```python
import json

with open("dataset_label_mapping.json") as f:
    mappings = json.load(f)

def get_legend_entries(dataset_key):
    """Return list of (class_name, hex_color) for the given dataset."""
    ds = mappings[dataset_key]
    # Prefer groups when present; fall back to the full class list.
    items = ds.get("groups") or ds["classes"]
    return [(item["name"], item["color"]) for item in items]
```

Inject the legend as an HTML overlay in the Leaflet map (absolute-positioned div).

### 4. Similarity Map Colorization

The similarity raster is UInt16 grayscale. Apply a diverging colormap (e.g., green-yellow-red) mapped to the display range 0-7,200:
- Low values (similar, no change) → green
- High values (dissimilar, changed) → red

### 5. Frontend UI Layout

The frontend has two main UI regions: a **sidebar** and the **map area**.

#### Sidebar

The sidebar is the primary control panel for layer selection. It presents two top-level layer groups:

1. **LULC layers** — one entry per dataset provider (Sunstone, Dynamic World, ESRI, etc.), each expandable to select a year
2. **Similarity layer** — single toggle (no year selector; it is always 2017 vs 2025)

Only one layer is active at a time. Selecting a layer from the sidebar switches the visible tile overlay on the map.

#### Legend

The legend is displayed as an overlay on the map and follows these rules:

- The legend is **only visible when a layer is selected/active**. If no layer is active, no legend is shown.
- **Datasets that share the same class schema share the same legend.** When the user switches between years within a single dataset (e.g., Sunstone 2017 → Sunstone 2020), the legend stays the same. When the user switches to a different dataset provider, the legend updates to reflect that dataset's classes and palette.
- For `glad_glclu` and `glc_fcs30d`, the legend shows the **simplified group** names and colors (not the full 100+ classes).
- The similarity layer has its own distinct legend (continuous colormap, not categorical classes).

#### Legend grouping by dataset

| Legend Group | Datasets sharing it |
|---|---|
| Sunstone 9-Class | `sunstone_kenya_lulc_9C` (all years) |
| Dynamic World | `dynamicworld` (all years) |
| ESRI LULC | `esri_lulc` (all years) |
| ESA WorldCover | `worldcover` (all years) |
| GLAD GLCLU (simplified) | `glad_glclu` (all years) |
| GLC_FCS30D (simplified) | `glc_fcs30d` (all years) |

#### Other controls

| Control | Purpose |
|---|---|
| **Opacity slider** | Adjust overlay transparency (0-100%) |
| **Click/pixel inspector** | On map click, query the raster value and display the class name + color |

### 6. Serving the Application

```python
import http.server
import socketserver

PORT = 8000
Handler = http.server.SimpleHTTPRequestHandler

with socketserver.TCPServer(("", PORT), Handler) as httpd:
    print(f"Serving at http://localhost:{PORT}")
    httpd.serve_forever()
```

The workflow:
1. Start `localtileserver` for each active raster (runs on its own port)
2. Generate the HTML map with `folium` (embeds tile URLs)
3. Serve the HTML via Python's HTTP server
4. Open browser to `http://localhost:8000/map.html`

## Development Conventions

- **File naming:** COG files follow `{dataset_key}_{year}_cog.tif` pattern. Do not rename data files.
- **COG creation:** Use `MODE` resampling for categorical rasters, `AVERAGE` for continuous:
  ```bash
  # Categorical (LULC)
  gdal_translate -of COG -co COMPRESS=LZW -co OVERVIEW_RESAMPLING=MODE -co BLOCKSIZE=512 input.tif output_cog.tif

  # Continuous (Similarity)
  gdal_translate -of COG -co COMPRESS=LZW -co OVERVIEW_RESAMPLING=AVERAGE -co BLOCKSIZE=512 input.tif output_cog.tif
  ```
- **CRS:** All rasters must be EPSG:4326. Reproject before adding to `data/`.
- **Label mappings:** All class definitions live in `dataset_label_mapping.json`. Never hardcode class names or colors in application code.
- **Basemap attribution:** Always include proper attribution for ESRI/OSM tiles.

## Key Implementation Notes

1. **localtileserver port management:** Each `TileClient` binds to a random port. For multi-raster support, track all active clients and their URLs. Consider using a single `TileClient` and swapping the file path when the user changes dataset/year.

2. **Large file handling:** The similarity raster is ~31,540 x 39,069 pixels. COG internal tiling + overviews handle this efficiently — always use COG format, never raw GeoTIFF.

3. **No-data handling:** Some `.tif_.gstmp` files in `data/` are incomplete downloads (DynamicWorld 2018/2020/2023/2024, WorldCover 2021). The application should gracefully skip or indicate unavailable years for these datasets.

4. **Year availability per dataset:**
   | Dataset | Available Years |
   |---|---|
   | sunstone_kenya_lulc_9C | 2017-2025 (all complete) |
   | dynamicworld | 2017, 2019, 2021-2022, 2025 (others incomplete) |
   | esri_lulc | 2017-2024 (all complete) |
   | worldcover | 2020 (2021 incomplete) |
   | glad_glclu | 2020 |
   | glc_fcs30d | 2019-2021 |
   | similarity | single file (2017 vs 2025) |
